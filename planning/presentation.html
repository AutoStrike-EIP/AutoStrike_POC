Architecture Hexagonale

Le server suit le pattern Clean Architecture :

  • domain/ = logique pure, pas de deps externes
  • application/ = orchestration des use cases
  • infrastructure/ = adapters (HTTP, DB, WS)

Fichiers cles a connaitre

  • server/cmd/autostrike/main.go
  • server/internal/domain/entity/
  • dashboard/src/lib/api.ts
  • agent/src/executor.rs

API REST - Endpoints Principaux

Base URL: https://localhost:8443/api/v1

Method Endpoint Description
GET /agents Liste tous les agents connectes
GET /techniques Liste toutes les techniques MITRE
GET /scenarios Liste tous les scenarios
POST /executions Lance une execution (scenario + agents)
GET /executions/:id/results Resultats d'une execution
GET /techniques/coverage Couverture MITRE par tactique
# Exemple: Lancer une execution curl -X POST https://localhost:8443/api/v1/executions \ -H "Content-Type: application/json" \ -d '{"scenario_id": "scenario-quick-scan", "agent_paws": ["abc123"], "safe_mode": true}'

Pourquoi Garder Cette Stack ?

"J'ai pose les fondations solides. Maintenant on construit dessus ensemble."

Go (Backend)

Kubernetes, Docker, Terraform, Caldera... tous en Go. C'est LE standard pour les outils d'infra et secu.

Rust (Agent)

Les agents Python se font bloquer par Windows Defender. Rust compile en natif, passe sous les radars.

React (Dashboard)

Framework le plus populaire. Tout le monde peut contribuer. TypeScript evite les bugs runtime.

L'avantage de l'architecture hexagonale

Changer SQLite ‚Üí PostgreSQL = 1 fichier. Changer le framework HTTP = 1 dossier. Le code metier ne bouge pas.
